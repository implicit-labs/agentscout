# Implicit

## Architecture
Monorepo with web client, iOS Capacitor client, Railway-deployed server, and daemon.
- Web client: packages/client/ (Next.js on port 3000)
- iOS client: packages/client-ios/ (Capacitor via Xcode, NOT TestFlight)
- Server: packages/server/ (deployed on Railway, auto-deploys on push)
- Daemon: packages/daemon/ (local process, logs at ~/.implicit/daemon.log)

## Build & Run
- Web: `pnpm dev` from packages/client/
- Server: pushes auto-deploy to Railway
- iOS: `xcodebuild -workspace ios/App/App.xcworkspace -scheme App -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 16'`
- Daemon: `node packages/daemon/daemon.js`

## Verification
After editing web UI (.tsx, .css):
1. Use Playwright to screenshot localhost:3000 and present inline
2. Use Playwright console_messages to check for errors

After editing iOS code (.swift):
1. Build with xcodebuild (see Build above)
2. Use ios-simulator MCP: screenshot, then ui_describe_all
3. For runtime logs: `xcrun simctl spawn booted log stream --predicate 'subsystem contains "implicit"' --timeout 10`

After pushing to feature branches:
1. Run `railway status` to check deployment
2. Run `railway logs --latest` for deploy errors
3. For daemon: `tail -n 50 ~/.implicit/daemon.log`

## Linear Integration
- Plans and specs MUST live as comments/descriptions on their Linear issue, not as local doc files
- When finding a local plan doc in docs/plans/, check Linear for the matching issue and sync
- Use the Linear MCP to search, create, and update issues

## Active Decisions
- iOS is deployed via Capacitor through Xcode to physical device, NOT TestFlight
- Hook relay goes through Railway endpoint at .up.railway.app
- Browser preference: use Arc for authenticated sessions (user is not logged in on Chrome)

## What NOT To Do
- Don't ask the user to "Hit Cmd+R in Xcode" — use xcodebuild directly
- Don't create local plan docs — put plans in Linear issues
- Don't guess at deployment status — use `railway status`
- Don't ask the user to paste logs — read daemon.log and use railway logs yourself
- Don't kill the web client the user started — there's no reason to restart it

NOTE: Hooks cannot call MCP tools directly — this hook outputs a reminder message that the agent should act on.

Preflight Check
✓ xcodebuild available — Xcode 26.0.1

Hook script 
#!/bin/bash
INPUT=$(cat)
CMD=$(echo "$INPUT" | jq -r '.tool_input.command // empty')
if echo "$CMD" | grep -q 'xcodebuild' && echo "$INPUT" | jq -r '.tool_output // empty' | grep -q 'BUILD SUCCEEDED'; then
  echo '{"message": "Build succeeded. Use ios-simulator MCP to screenshot and ui_describe_all to verify the UI before reporting done."}'
fi
settings.json entry 
{
  "event": "PostToolUse",
  "matcher": "Bash",
  "command": "~/.claude/hooks/ios-auto-verify.sh",
  "timeout": 10
}
